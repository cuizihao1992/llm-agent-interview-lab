Interview Playbook

面试回答节奏

01

先点出核心矛盾

例如不能全量塞 prompt、RAG 不是只接向量库、长上下文不能替代数据治理。

02

再拆系统结构

用分层记忆、数据管线、召回重排、KV Cache 管理等结构承接问题。

03

补工程取舍

讲成本、延迟、权限、索引、并发、评估指标，面试回答会更像做过系统。

04

收束成趋势判断

例如 Long RAG、异步记忆更新、混合检索、算子和架构联合优化。

Build Path

后续开发路线

  1. 文档解析与切片 实现 Markdown / PDF / 网页解析、语义切片、父子块结构。
  2. 检索与重排 接入 Embedding、向量库、关键词检索、reranker 和召回评估。
  3. Agent 记忆 实现短期对话缓存、结构化画像、长期情景记忆和异步更新。
  4. 面试训练 Demo 基于知识库做问答、引用来源、回答评分和知识盲区提示。
