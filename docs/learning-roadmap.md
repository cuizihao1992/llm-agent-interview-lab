# 学习路线

## 阶段 1：先掌握面试表达框架

目标是能在 2 到 3 分钟内讲清每个问题的核心矛盾、方案拆解和工程取舍。

建议顺序：

1. 长期陪伴型 AI 角色的记忆机制设计。
2. RAG 向量数据全流程工程化方案。
3. 长上下文模型是否会取代 RAG。
4. Transformer 处理超长上下文的瓶颈与优化方案。

## 阶段 2：补齐工程细节

每个模块都要能落到实际代码或系统设计：

- 记忆机制：短期对话缓存、实体画像抽取、情景记忆向量召回、冲突更新。
- RAG：解析、清洗、切片、向量化、索引、召回、重排、评估。
- 长上下文：成本、延迟、中间迷失、KV Cache、位置编码。
- 优化技术：FlashAttention、GQA、PagedAttention、RoPE Scaling、ALiBi。

## 阶段 3：做一个最小 Demo

推荐最小闭环：

1. 准备一批 Markdown 面试知识文档。
2. 对文档做切片和元数据提取。
3. 用 Embedding 模型生成向量。
4. 用向量库召回相关片段。
5. 把召回片段和用户问题拼成提示词。
6. 让大模型生成带引用的回答。

## 阶段 4：做面试级项目包装

项目描述可以这样表达：

> 我构建了一个面向大模型 Agent 面试知识的本地 RAG 学习系统。它支持文档解析、语义切片、向量化、混合检索、重排序和引用回答。后续我还扩展了 Agent 记忆模块，把短期上下文、结构化用户画像和长期情景记忆分层管理，用异步任务做记忆抽取、冲突合并和遗忘更新。
